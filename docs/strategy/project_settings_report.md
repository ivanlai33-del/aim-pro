## 1. 訂閱方案與定價策略 (Subscription & Pricing)

> *   **17 種商業模組 (Modules)**：這才是真正的訂閱標的。每個模組代表一顆獨立的「專業大腦」（如「品牌設計大腦」、「SEO 策略大腦」）。用戶是針對這些 Specific 的大腦進行訂閱。

## 2. 使用者權限與角色分配 (Roles & Permissions)

系統目前在 `useModuleAccess.ts` 中定義了三層權限檢查邏輯：

1.  **開發者模式 (Dev Mode)**：
    *   **權限**：無敵模式，解鎖所有功能與核心 AI Prompt。
    *   **用途**：內部開發與除錯使用。
2.  **系統管理員 (Admin)**：
    *   **權限**：可管理所有用戶訂閱狀態、查看全站流水帳 (Dashboard)、設定全域參數。
    *   **限制**：無法查看用戶的私有合約內容 (隱私保護)。
3.  **一般用戶 (User)**：
    *   **權限**：依據 `SubscriptionTier` 動態解鎖對應的行業模組。
    *   **限制**：僅能存取自己建立的專案與財務資料。

## 3. 系統核心機制 (Core Mechanics)

### A. AI 混合模式 (Hybrid AI Model)
*   **託管模式 (Managed)**：使用平台提供的 AI 額度 (受 Rate Limiting 限制)。
*   **自訂金鑰 (BYOK)**：允許用戶輸入自己的 OpenAI API Key，解鎖無限速生成與更長的 Context Window。

### B. 角色人格設定 (AI Persona)
*   **核心人格 (Core Persona)**：由系統鎖定 (System Managed)，確保 AI 輸出的專業度與法律合規性。
*   **自定義指令 (Custom Overlay)**：用戶可疊加個性化指令 (e.g., "語氣更活潑"、"強調售後服務")，系統會將其透過 Prompt Engineering 融合。

### C. 資料流與隱私
*   **本地優先**：用戶的 API Key 與部分敏感草稿優先儲存於 LocalStorage。
*   **雲端同步**：正式報價單與合約寫入 Supabase (透過 RLS 權限隔離)。
